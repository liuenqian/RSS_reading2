# 来源内 Excel 式初筛表设计

日期：2026-07-18
状态：设计已确认，待规格审查与用户书面复核

## 1. 背景与目标

Cento 已经为 PubMed 检索结果和 RSS 订阅结果提供卡片式阅读、标星、已读、标签、阅读笔记、期刊指标、筛选、排序、初筛状态和 XLSX 导出。大量文献的第一轮筛选更适合高密度表格：用户需要在同一屏幕比较标题、摘要、年份、期刊指标和筛选状态，并快速完成标星、保留、待定、排除、标签与笔记操作。

本功能在每个 PubMed 检索和 RSS 订阅内部增加“表格初筛”视图。它不是独立复制文献，也不是要求用户在外部 Excel 中完成全部操作；表格与现有卡片视图、右侧详情和数据库状态共享同一份文献数据。同时支持将当前来源导出为可编辑 XLSX，并把初筛工作字段安全导回 Cento。

第一版以单个来源为边界。底层使用通用“初筛范围”接口，为后续把多个来源合并为独立初筛项目预留扩展点，但本次不实现项目管理和多来源合并界面。

## 2. 已确认的产品原则

1. PubMed 检索和 RSS 订阅均可切换到 Excel 式表格，不创建第二份文章记录。
2. 第一版采用来源内表格，每个检索或订阅独立保存列、筛选、排序和行高配置。
3. 标星、已读、标签和阅读笔记属于全局文献状态，在卡片、表格及不同来源之间同步。
4. 初筛状态、排除原因和筛选备注属于来源状态；同一篇文章在不同来源中可以有不同判断。
5. 表格支持隐藏列、显示列、调整列顺序、调整列宽、固定关键列和恢复默认布局。
6. 表格支持行排序、升降序和多级排序；排序与列顺序是两个独立概念。
7. 默认使用紧凑表格加右侧详情；可切换摘要行高模式，并按来源保存。
8. XLSX 支持来源全部、当前筛选结果和当前勾选文献三种范围，并可安全导回工作字段。
9. Excel 导入不覆盖标题、作者、期刊、PMID、DOI 等文献元数据。
10. Cento 与 Excel 同时修改同一状态时，默认保留 Cento 当前值；用户可在导入预览中逐项采用 Excel。
11. 表格组件只依赖“初筛范围类型 + 范围 ID”，预留 `project` 范围类型，不将实现写死为 PubMed 或 RSS。

## 3. 范围

### 3.1 本次包含

- PubMed 检索结果的“卡片阅读 / 表格初筛”视图切换。
- RSS 订阅结果的“卡片阅读 / 表格初筛”视图切换。
- 来源级表格布局、列状态、过滤状态、排序状态和行高状态持久化。
- 高密度表格、固定表头、横向滚动、列宽调整和关键列固定。
- 直接编辑标星、已读、标签、初筛状态、排除原因和筛选备注。
- 点击表格行后在现有右侧详情区快速阅读完整文献。
- 多选与批量状态操作。
- 来源级 XLSX 导出、导入预览、冲突处理和事务写回。
- 大数据量的增量加载或虚拟滚动。
- 通用初筛范围接口，为后续独立项目预留扩展能力。

### 3.2 本次不包含

- 多个 PubMed/RSS 来源合并为一个独立初筛项目。
- 独立项目的新建、重命名、归档或成员管理界面。
- 实时监听外部 XLSX 文件变化。
- 与 Microsoft Excel、Google Sheets 或其他云表格服务双向在线同步。
- 通过 Excel 修改英文标题、摘要、作者、期刊、PMID、DOI 或期刊指标。
- 在表格中实现完整电子表格公式、跨单元格计算或宏。
- 改变现有卡片阅读和右侧详情的核心信息结构。

## 4. 初筛范围模型

### 4.1 通用范围标识

前后端统一使用初筛范围：

```text
scope_kind: pubmed | feed | project
scope_id:   对应来源或项目的稳定数字 ID
```

第一版只允许 `pubmed` 和 `feed`。`project` 作为保留类型出现在接口、数据库约束和 XLSX 格式设计中，但没有可见入口。

表格组件通过范围接口加载：

- 范围元信息：名称、类型、文献总数和更新时间。
- 范围成员：规范 `entry_id` 及其来源级初筛状态。
- 文章信息：标题、译文、摘要、作者、期刊、年份、标识符和指标。
- 用户工作状态：标星、已读、标签、阅读笔记和来源级筛选字段。

未来独立项目只需要提供项目成员关系和相同的范围读取接口，不需要复制表格组件、筛选器或 XLSX 逻辑。

### 4.2 状态归属

全局文献状态继续附着于规范文章 `entry_id`：

- 标星
- 已读
- 标签
- 阅读笔记

来源级状态附着于 `scope_kind + scope_id + entry_id`：

- 初筛状态：未筛选、保留、待定、排除
- 排除原因
- 筛选备注
- 最后筛选时间

全局状态在任何来源中修改后立即同步。来源级状态只影响当前检索或订阅，避免同一文章在不同研究问题中被迫共享判断。

现有标星保存在前端 `localStorage`，不能参与数据库事务。第一版必须先将标星迁入数据库中的全局文章状态，再启用表格导入写回；不能保留一半数据库、一半浏览器存储的双写状态。

## 5. 页面与交互

### 5.1 入口与视图切换

在现有文献列表头部增加紧凑的视图分段控制：

`卡片阅读 | 表格初筛`

切换只改变中间文献列表的呈现方式，不切换来源、不清空筛选、不关闭右侧详情。应用按来源记住最后一次使用的视图。

已有卡片视图保持原行为。表格视图复用现有顶部来源标题、更新、导出和批量动作，不创建第二套页面级导航。

### 5.2 表格结构

默认列分为四组：

| 分组 | 默认列 |
| --- | --- |
| 操作 | 勾选、标星、编号、初筛状态 |
| 文献 | 标题、标题中文、摘要、摘要中文、作者、期刊、发表时间 |
| 指标 | IF、JCR Q、中科院分区、Top |
| 工作 | 已读、标签、排除原因、筛选备注、阅读笔记 |

PMID、PMCID、DOI、来源加入时间等辅助列默认隐藏，可在列设置中开启。

表头固定在滚动区域顶部。表格具有稳定列宽与最小宽度；宽表使用横向滚动，不压缩到无法阅读。标题、摘要和笔记单元格采用文本截断与明确行高，不允许动态内容导致整张表跳动。

### 5.3 固定列

默认固定左侧关键列：

- 勾选
- 标星
- 编号
- 初筛状态
- 标题

用户可以在列设置中取消或增加固定列。固定列必须形成从左到右连续区域，避免多个分散粘滞层产生遮挡和复杂滚动行为。

### 5.4 列设置

表格工具栏提供带图标的“列设置”菜单：

- 复选框控制显示或隐藏。
- 拖动手柄调整列顺序。
- 固定/取消固定列。
- 恢复默认列。

列宽通过表头分隔线拖动调整。列宽、显示状态、顺序和固定状态在操作后防抖保存到当前来源配置。

每个 PubMed 检索和 RSS 订阅拥有独立配置。切换来源时立即恢复该来源的布局，不把上一个来源的临时状态覆盖到新来源。

### 5.5 行高模式

工具栏提供行高模式：

- `紧凑`：默认。标题最多两行，摘要不在表格中展开，适合高密度第一轮扫描。
- `摘要`：标题完整显示，摘要最多显示约 4–6 行，适合以摘要为主的第二轮筛选。

完整摘要、PDF、标签和阅读笔记始终可在右侧详情查看。行高模式按来源保存。

### 5.6 快速阅读

单击行或标题后：

- 当前行进入选中状态。
- 右侧沿用现有详情面板显示完整文献。
- 切换行时详情随之更新，不重建整张表。
- 当前行在筛选、排序和状态更新后尽量保持可见。

表格单元格内的按钮、复选框、菜单和输入控件阻止行点击冒泡，避免标星或修改状态时误切换详情。

### 5.7 直接编辑与批量操作

表格直接支持：

- 星形图标切换标星。
- 复选框或状态控件切换已读。
- 下拉菜单修改初筛状态。
- 标签菜单增删标签。
- 排除原因和筛选备注的紧凑编辑器。
- 阅读笔记入口打开现有笔记编辑体验。

多选后支持批量设置初筛状态、标签、标星和已读。批量操作显示受影响数量并复用现有批量选择状态，取消操作时不修改数据。

## 6. 筛选与排序

### 6.1 筛选

表格复用并扩展现有筛选能力：

- 关键词：英文标题、中文标题、英文摘要、中文摘要、作者、期刊。
- 初筛状态：未筛选、保留、待定、排除。
- 标星与已读。
- 发表年份或日期范围。
- IF 数值范围。
- JCR Q、中科院分区和 Top。
- 标签。
- 排除原因是否为空。
- 是否存在阅读笔记。

筛选条件组合使用 AND 语义；同一个多选筛选器内部使用 OR 语义。筛选摘要显示当前完整来源中的命中数量，并提供一键清空。命中数量、全选和导出不得以当前已加载或已渲染的行作为数据源。

### 6.2 行排序

行排序与列顺序完全分离。支持以下主要字段：

- 发表时间
- 加入来源时间
- IF
- JCR Q
- 中科院分区
- Top
- 标星
- 已读
- 初筛状态
- 标题
- 作者
- 期刊

排序支持升序、降序和最多三个层级的多级排序。缺失值稳定排在末尾，现有期刊指标排序规则继续复用。相同排序值使用来源内稳定编号作为最后比较键，保证重新渲染时顺序不抖动。

### 6.3 来源级配置

每个范围独立保存：

- 当前视图
- 列显示与顺序
- 列宽与固定列
- 行高模式
- 筛选条件
- 多级排序
- 每页数量或已加载数量
- 表格纵向和横向滚动位置

配置记录带格式版本。字段新增、删除或改名时通过版本迁移合并默认列，不能因为旧配置缺少新字段而永久隐藏新能力。

## 7. 数据持久化

### 7.1 表格配置

新增来源级配置存储，逻辑键为：

```text
scope_kind + scope_id
```

配置内容使用结构化 JSON 或等价类型化模型，至少包含：

- `schema_version`
- `view_mode`
- `row_density`
- `columns[]`: key、visible、order、width、pinned
- `filters`
- `sorts[]`
- `render_limit`
- `scroll_top`
- `scroll_left`
- `updated_at`

后端校验允许的列键、排序字段、宽度范围和数组数量，避免损坏配置导致页面无法打开。无效片段回退默认值，而不是拒绝整个来源。

### 7.2 全局状态数据库迁移

新增数据库级全局文章状态，用于持久化标星及其更新时间。现有已读、标签和阅读笔记继续使用各自数据库表，但通过统一服务参与表格读取和事务写入。

标星迁移流程：

1. 前端读取现有 `starred-ids`。
2. 调用幂等后端迁移命令，将仍存在的 `entry_id` 合并写入数据库标星状态。
3. 后端返回已迁移与未知 ID 数量。
4. 只有后端事务提交成功后，前端才写入迁移完成标记并移除旧 `starred-ids`。
5. 迁移完成前读取使用数据库标星与旧 ID 的并集；写入只走数据库，避免迁移期间丢失新操作。

迁移命令可以重复执行，不因应用中断、重复启动或旧 ID 指向已删除文章而失败。卡片视图、筛选计数和表格必须改为读取同一个数据库标星来源。

### 7.3 来源级筛选状态

PubMed 已有的来源级初筛状态应通过通用服务访问。RSS 来源需要新增按 `feed_id + entry_id` 保存的来源级状态记录。迁移不得把已有 PubMed 初筛结果改成全局状态，也不得删除现有判断。

RSS 迁移按当前订阅成员关系回填：

1. 遍历每个 RSS 订阅与其文章成员关系。
2. 对每个 `feed_id + entry_id`，复制该文章在旧 `entry_screening_status` 中的状态和时间；没有旧记录时使用“未筛选”。
3. 同一文章属于多个订阅时，每个订阅得到相同的初始副本，迁移后允许独立修改。
4. 已经存在的新来源级记录不被重复迁移覆盖。
5. 旧 `entry_screening_status` 暂时保留给“全部文章”等非来源视图和降级兼容；来源内 RSS 表格只读取新记录。

迁移和建表在同一数据库迁移事务中完成。迁移测试必须覆盖一篇文章属于零个、一个和多个订阅的情况。

通用服务负责：

- 读取范围成员及来源级状态。
- 更新单条初筛状态。
- 批量更新范围内成员。
- 校验文章属于指定范围。
- 为导出和导入提供字段级基线值。

### 7.4 并发与字段级基线

XLSX 导出为每个可写字段记录导出时的规范化基线值，而不是只记录一个全局版本和一个来源版本。导入时对“导出基线、Cento 当前值、Excel 候选值”进行三方比较，识别具体字段是否冲突。

基线字段包括：标星、已读、标签集合、初筛状态、排除原因和筛选备注。标签按去重、去空、稳定排序后的数组比较；布尔值和状态使用固定代码；文本统一换行但不折叠有意义的正文空格。

各数据库记录仍保留 `updated_at` 用于审计和增量刷新，但 XLSX 冲突判断以字段级基线值为准，避免修改标签导致标星或初筛状态被误判冲突。

## 8. XLSX 导出

### 8.1 入口和范围

表格工具栏继续复用现有“导出”流程，并增加“初筛表 XLSX”用途。用户选择：

- 当前来源全部
- 当前筛选结果
- 当前勾选文献

导出列默认跟随当前表格可见列，但用户可在导出弹窗中独立调整，不改变屏幕布局。工作字段和识别字段始终包含，必要的内部列可以隐藏。

### 8.2 可见列

用户可见工作簿至少支持：

- 编号
- PMID、PMCID、DOI
- 英文标题、中文标题
- 英文摘要、中文摘要
- 作者、期刊、发表时间
- IF、Q、B、Top
- 初筛状态、排除原因、筛选备注
- 标星、已读、标签、阅读笔记汇总（只读）

工作字段提供数据验证：初筛状态、是/否字段使用下拉或受限值，减少手工输入差异。

一篇文章可以存在多条按 `profile_id` 区分的阅读笔记。第一版将这些笔记以带笔记名称的只读汇总文本导出，供 Excel 初筛时参考；导入器忽略该列。用户需要修改笔记时继续使用 Cento 右侧详情中的现有笔记编辑器，避免一个单元格覆盖多条结构化笔记。

### 8.3 隐藏识别列

XLSX 包含不可依赖中文表头解析的稳定内部列：

- 格式代码，例如 `CENTO_SCREENING_1`
- `scope_kind`
- `scope_id`
- `entry_id`
- PMID/DOI 校验值
- 每个可写字段的导出基线值

导入器按格式代码和固定内部列识别文件，不依赖文件名、工作表名称、列顺序或可见表头文字。

基线值放在隐藏工作表或隐藏内部列中，并与 `entry_id` 一一对应。标签基线使用规范 JSON 数组；文本字段使用纯文本；内部值不使用公式。

## 9. XLSX 导入与冲突处理

### 9.1 允许写回的字段

导入只允许修改：

- 标星
- 已读
- 标签
- 初筛状态
- 排除原因
- 筛选备注

标题、摘要、作者、期刊、发表时间、PMID、DOI、期刊指标和阅读笔记汇总只用于人工阅读与身份校验，永不从该初筛 XLSX 覆盖数据库。

### 9.2 导入预览

导入前显示：

- 文件来源和当前来源是否匹配。
- 可更新、无变化、未知文献、重复行、非法值和冲突数量。
- 每个冲突字段的 Cento 当前值与 Excel 候选值。
- 将采用 Cento 或 Excel 的最终选择。

来源不匹配时默认阻止导入，不允许仅凭 PMID 猜测导入到另一个检索或订阅。未来独立项目需要显式的“复制为新项目”流程，不复用普通导入绕过范围边界。

### 9.3 冲突规则

每个字段独立执行三方比较，设导出基线为 `B`、Cento 当前值为 `C`、Excel 候选值为 `E`：

- `E = B`：Excel 没有修改该字段，不写入。
- `C = B` 且 `E != B`：只有 Excel 修改，安全采用 Excel。
- `C = E`：两端结果一致，不产生冲突，也无需重复写入。
- `C != B`、`E != B` 且 `C != E`：双方修改为不同值，标记字段冲突。

发生字段冲突时：

- 默认保留 Cento 当前值。
- 预览允许逐项切换为“采用 Excel”。
- 可对同类冲突批量选择，但必须显示受影响数量。
- 不允许一个总开关静默覆盖所有字段。

不同字段独立比较。例如 Cento 更新了标签、Excel 更新了初筛状态，两项都可以安全合并，不构成整行冲突。

### 9.4 事务写入

确认导入后，所有通过校验的更改在一个数据库事务中写入。标星在启用此流程前已经迁入数据库，因此标星、已读、标签和来源状态可以与其他数据库工作字段一起原子写入。任一写入失败时整批回滚，预览内容保留以便重试。

事务提交后，仅增量更新受影响行、筛选计数和右侧详情；不重建整个来源列表，也不重置当前滚动位置。

## 10. 性能设计

表格面向数千至上万条来源成员，必须避免一次创建全部 DOM 行。实现采用后端完整结果查询加前端窗口化渲染，不允许以现有 RSS `LIMIT 200` 或当前 `render_limit` 作为逻辑数据集边界。

范围查询服务接收结构化筛选和排序规格，返回总数、分页行和稳定排序位置。筛选、排序、全选、批量操作和导出必须复用同一套范围查询语义：

- 页面只请求当前窗口所需的行和少量缓冲。
- “当前筛选结果”由后端根据完整来源成员计算，不由 DOM 或已加载数组计算。
- “全选当前筛选结果”保存为筛选规格加排除 ID，而不是把已加载行 ID 当作全部选择。
- 批量操作在执行时由后端重新解析该选择描述并校验范围。
- XLSX 导出在后端使用相同筛选和排序规格，保证导出数量和界面计数一致。
- 行编号使用完整排序结果中的位置，不因分页从 1 重新开始。

窗口化渲染要求：

- 只渲染可见区域及少量缓冲行。
- 排序和筛选在范围查询服务完成，不读取 DOM 文本。
- 摘要模式仍只渲染可见摘要。
- 状态更新只替换目标行模型与单元格。
- 列宽拖动期间不持续写数据库，结束后防抖保存。
- 搜索输入防抖，并通过请求序号丢弃过时结果。

第一版应针对至少 10,000 条模拟文献验证初次打开、滚动、排序、筛选和批量选择的可用性。

## 11. 错误处理

- 来源没有文献：显示空状态，不渲染空表头工具。
- 表格配置损坏：保留有效配置片段，其余回退默认布局，并允许恢复默认。
- 列配置引用未知字段：忽略未知字段并追加新默认列。
- 当前行因筛选消失：保留右侧详情，明确该文献不在当前筛选结果中。
- 状态保存失败：恢复单元格旧值并显示简短错误，不假装保存成功。
- 批量操作部分校验失败：写入前拒绝整批，列出不属于当前范围的文献。
- XLSX 格式不支持：拒绝导入，不按表头猜测。
- XLSX 来源不匹配：阻止写入并显示导出来源和当前来源。
- Excel 枚举值非法：标记具体行和字段，不将未知文字映射为默认状态。
- Excel 重复行：标记冲突，默认不写入重复项。
- 导入事务失败：整批回滚。

## 12. 组件边界

### 12.1 前端

- `ScreeningTable`：表格渲染、虚拟滚动、选择和行内操作。
- `ScreeningToolbar`：视图、筛选、排序、列设置、行高和导出入口。
- `ScreeningColumnManager`：列可见性、顺序、固定和恢复默认。
- `ScreeningImportPreview`：导入分类、字段冲突选择和确认。
- 范围适配层：把 PubMed、RSS 和未来项目转换为统一表格模型。

具体文件划分遵循现有前端架构；若继续在单一 `main.js` 中增加完整表格会显著扩大耦合，应将纯排序、列配置、范围键和 XLSX 预览模型放入可单测模块，主文件只负责页面编排与现有状态桥接。

### 12.2 后端

- 范围服务：校验范围、列出成员和读取来源级状态。
- 表格配置服务：校验并持久化来源级配置。
- 初筛状态服务：单条和批量更新来源状态。
- 初筛 XLSX 服务：生成、解析、预览、冲突分类和事务写入。

XLSX 服务不直接耦合 PubMed 查询或 RSS 抓取逻辑，只通过范围服务获取授权成员。

## 13. 测试与验收

### 13.1 后端测试

- PubMed 与 RSS 范围正确返回成员，并拒绝越权 `entry_id`。
- 同一文章在两个来源中的初筛状态互不影响。
- 标星、已读、标签和笔记仍是全局状态。
- 来源级表格配置读写、默认值、版本迁移和损坏回退。
- 批量状态更新在校验失败时不部分写入。
- XLSX 三种范围导出无遗漏、无越界。
- 隐藏识别列可在可见列重排、改名后继续解析。
- 只允许工作字段写回，元数据永不覆盖。
- 标星从 `localStorage` 幂等迁入数据库，迁移中断后重试不丢失状态。
- RSS 旧初筛状态按每个订阅成员关系回填，同一文章在多个订阅中得到独立副本。
- 阅读笔记汇总在 XLSX 中只读，导入不会修改或合并笔记。
- 字段级三方比较只标记真正冲突的字段。
- 默认保留 Cento，显式选择后采用 Excel。
- 导入失败时事务整体回滚。

### 13.2 前端测试

- PubMed 和 RSS 均显示卡片/表格视图切换。
- 每个来源独立恢复视图、列、宽度、固定、行高、筛选和排序。
- 隐藏列和列重排不改变行数据。
- 多级排序稳定，缺失指标位于末尾。
- 行内标星、已读和状态更新不误触发行切换。
- 点击行更新右侧详情，且不重建整张表。
- 批量操作显示正确数量并更新目标行。
- 导入预览按字段显示冲突与最终策略。
- 用户取消导入时不修改数据库。
- 当前筛选计数、全选、批量操作和导出覆盖完整来源，而非当前已加载窗口。

### 13.3 性能测试

- 10,000 条模拟文献不会创建 10,000 个同时存在的行节点。
- 后端对 10,000 条完整来源返回准确筛选总数和稳定分页。
- 连续滚动无明显空白闪烁或布局跳动。
- 排序和筛选后保持稳定编号和选中行。
- 摘要模式只渲染可见范围的长文本。
- 单条状态更新不触发全量排序，除非修改字段正是当前排序或筛选条件。

### 13.4 验收场景

1. 打开一个 PubMed 检索，切换到表格初筛。
2. 隐藏摘要英文列，调整标题中文列位置和宽度，切换摘要行高。
3. 按 IF 降序、年份降序进行多级排序，并筛选未筛选、Q1/Q2、未读文献。
4. 快速标星、设置保留/待定/排除，点击行在右侧阅读完整摘要。
5. 切换到另一个 RSS 订阅，确认其列与筛选配置独立。
6. 返回原 PubMed 检索，确认布局、排序、筛选、行高和滚动位置恢复。
7. 导出当前筛选结果，修改标星、已读、标签和初筛状态；阅读笔记汇总保持只读。
8. 在 Cento 中同时修改一条状态，然后导入 XLSX。
9. 预览正确显示字段级冲突，默认保留 Cento，手动指定一项采用 Excel。
10. 确认后，卡片视图、表格和右侧详情显示一致状态。

## 14. 后续独立初筛项目

后续版本可以增加：

- 从一个或多个 PubMed/RSS 来源创建命名项目。
- 以规范 `entry_id` 去重合并来源成员。
- 保存文章来自哪些检索或订阅。
- 冻结项目批次并记录新增文章。
- 复用本设计的表格配置、来源级状态、XLSX 和冲突处理。

项目范围使用已预留的 `scope_kind = project`。第一版不得提前加入没有用户入口的复杂项目管理逻辑，但所有公共接口、识别列和组件命名应避免只能表示 PubMed 或 RSS。
